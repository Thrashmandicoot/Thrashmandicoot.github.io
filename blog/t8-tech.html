A Quick Overview

Active Record takes a database table and wraps it in a class. Each instance of a class object is represented as a row on the table. This in turn gives the programmer the ability to use specific methods to modify and query against the tables that are being used across their program.

Lets take a look at active record in action - say you wanted to build a blog, and that blog has posts, and those posts have comments. We could easily setup this posts object within Rails.

For example, say we generated a posts table using rails, and we gave it attributes such as title and body (for the title of the post and the text in that post respectively), this posts table would then inherit from ActiveRecord ( Posts < ActiveRecord::Base) and gain various methods that allow you to query against that table, and to add additional attributes to that table and associate that table with other tables.

So now we have a posts table, but what if we want to represent comments? We could easily use rails to generate a new comments table and associate those comments to our posts. Our comments would have active record methods available to them as well, and we could quickly modify that data.

For instance, say we had a comment we wanted to remove, comments.destroy() would allow us to kill that comment (say if we wanted to tie that comment to a delete button that we later gave control of to a user), and this would be the “Destroy” part of CRUD (fully known as create, read, update and destroy).

What about create? Create would allow us to easily create a new post, for instance, say we called posts.create( :title => “Hello World”, :body => “Welcome to our website!”) this would create a new instance of our posts object that would get an ID (auto generated) a title (“Hello World”) and body (“Welcome to our website!”) which would then be represented on our table.

Since we've created a post, we can now access or read that information, which is the R portion of CRUD.

Update is what allows us to update our posts information, say if we wanted to change the title, or add change functionality as part of one of our website's functions and destroy we've already covered.

This concludes our quick overview of Active Record. Hope you enjoyed!
